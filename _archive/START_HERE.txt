Welcome! This is a complete, ready-to-deploy referee scheduling system.

🎯 WHAT THIS DOES
═══════════════════
✓ Referees submit availability through a web form
✓ Data automatically syncs to Airtable database
✓ Admins filter and assign referees to games
✓ Track referees, clubs, fields, games, and availability
✓ Mobile-friendly, easy to use, no coding required


⚡ QUICK START (30 minutes total)
════════════════════════════════════

Step 1: READ THE GUIDE (5 min)
   📄 Open: QUICK_START.md
   This walks you through everything step-by-step

Step 2: SET UP AIRTABLE (15 min)
   📄 Follow: SETUP_GUIDE.md
   - Create free account at airtable.com
   - Create 5 tables (Referees, Clubs, Fields, Games, Availability)
   - Get your API credentials

Step 3: CONFIGURE (5 min)
   ⚙️ Windows: Double-click create-config.bat
   ⚙️ Mac/Linux: Run ./create-config.sh
   - This creates config.js from the template
   - Edit config.js and add your Airtable credentials

Step 4: TEST (5 min)
   🌐 Double-click: index.html
   - Opens in your web browser
   - Fill out the form
   - Check Airtable to see your submission

Step 5: DEPLOY (5 min)
   🚀 Easiest method:
   - Go to netlify.com/drop
   - Drag and drop this entire folder
   - Get your live URL instantly!
   - Share with referees

DONE! 🎉

🗄️ DATABASE STRUCTURE
══════════════════════

5 interconnected tables in Airtable:

1. REFEREES
   └─ Contact info, certifications, preferences

2. CLUBS
   └─ Club details, assignors, regions

3. FIELDS (Locations)
   └─ Field addresses, parking, facilities

4. GAMES
   └─ Date, time, teams, assigned referees

5. AVAILABILITY
   └─ Form submissions from referees


🎯 EXAMPLE WORKFLOW
═══════════════════

1. Referee visits your website
2. Fills out availability form:
   - Date available
   - Time range
   - Preferred locations
   - Positions willing to work
3. Clicks "Submit"
4. Data appears instantly in Airtable
5. Admin filters available referees:
   - Date = Saturday
   - Location = Rye
   - Position = Center Ref
6. Admin assigns referee to game
7. Everyone sees the assignment in Airtable

To use this system you need:

🔐 IMPORTANT: CONFIGURATION
═══════════════════════════

Before using, you MUST:
  1. Copy config.template.js to config.js
  2. Add your Airtable API key
  3. Add your Airtable Base ID

Use the helper scripts:
  Windows: create-config.bat
  Mac/Linux: create-config.sh

Never commit config.js to public repositories!
